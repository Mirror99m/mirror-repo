# Godot 4 2D平台动作游戏开发教程

> 本教程基于 Single-Minded Ryan 的 Godot 4 平台动作游戏实战课程（2025年适配 Godot 4.4），并依据你的实际代码进行了细节补充与修正。

## 1. 创建 TileMap 节点


### 介绍
在 Godot 中，**TileMap** 是用于高效创建 2D 游戏世界的节点。它使用一张包含多个小图块的图集（称为 **TileSet**），像铺地砖一样在网格上快速“盖”出关卡。

主要特点：
- **高性能**：相比使用成百上千个独立 `Sprite` 节点，TileMap 将所有图块合并为少数绘制指令，大幅降低 CPU/GPU 开销。
- **高效编辑**：提供画笔、填充桶等工具，直接在编辑器视图中绘制地形，无需逐个创建和摆放节点。
- **内置物理与导航**：可在 TileSet 中为每个图块定义碰撞形状、导航多边形（AI 寻路）、遮挡效果（Y-Sort），绘制时自动生效。
- **自动铺砌（AutoTile）**：通过配置地形系统，图块可根据相邻关系自动变化（如草地边缘自动过渡到泥土），加快自然风格关卡的搭建速度。

> **Godot 4 变化**：原来的 `TileMap` 节点被拆分为更灵活的 `TileMapLayer`。你可以使用单个 `TileMapLayer`，也可叠加多个层实现复杂地图结构。

### 项目设置
1. 创建根节点 `Node2D`，在项目根目录下新建 `game` 文件夹。
2. 在 `game` 文件夹中创建新场景 `main.tscn`。
3. 将 `main.tscn` 设为主场景：**项目 → 项目设置 → 运行 → 主场景**，选择 `res://game/main.tscn`。
   - 按 F5 运行游戏时，无论当前打开哪个场景，都会从主场景启动。
   - 导出游戏时，也将从主场景开始运行。

### 添加 TileMap 节点
1. 在 `main.tscn` 的根节点 `Node2D` 下，创建 `TileMapLayer` 节点（或直接添加 `TileMapLayer`）。
2. 在右侧 **Tile Set** 属性中，点击 **新建 TileSet**。
3. 展开 **地形集（Terrain Sets）**，点击 **添加元素**，将 Mode 设为 **匹配边和角（Match Corners and Sides）**。
4. 点击 **添加图块（Add Tiles）** 按钮，选择你的瓦片图片资源（如 `tileset.png`），并确保 **图块尺寸** 与图片中每个瓦片的像素尺寸一致（例如 16×16）。
5. 将瓦片资源拖入图块列表中，设置地形集为 0。
6. 绘制关卡：选中图块后，在场景视图里点击网格即可放置。可使用画笔、矩形、填充等工具。
7. 如果瓦片显示模糊，在 **渲染 → 纹理** 中调整过滤模式为 **最近邻（Nearest）**（适用于像素风格）。

> 💡 **小提示**：如果使用 `TileMapLayer`，可以创建多个层（如地面层、装饰层、碰撞层），方便管理。

## 2. 创建玩家节点

### 添加角色节点
1. 在 `main.tscn` 根节点下，添加 **CharacterBody2D** 节点，命名为 `Player`，这是玩家角色的物理刚体，用于处理移动和碰撞。并在脚本中使用 `class_name PlayerController`，方便其他节点引用。
2. 为 `Player` 添加子节点 **CollisionShape2D**，形状选择 **胶囊（CapsuleShape2D）**，调整大小匹配角色外观。
3. 添加子节点 **AnimatedSprite2D**，命名为 `AnimatedSprite2D`（默认名称即可），用于播放角色动画。
   - 在 **SpriteFrames** 中新建精灵帧资源将角色的动画帧图集（如 `player.png`）拖入，设置 **水平帧数** 和 **垂直帧数** 以分割动画，根据动画表设置帧数。
   - 为不同动作创建动画：`Idle`、`Run`、`Jump`、`Hit_Stand`、`Hit_Jump`、`Die`、`Shoot_Stand`、`Shoot_Run`、`Shoot_Jump`（名称与代码中使用的 `animated_sprite_2d.play()` 参数完全一致）。
4. 添加子节点 **Node2D**，命名为 `Shooting_Point`，作为子弹发射原点，放置在角色枪口位置（代码中根据方向设置 x = ±26，y 为肩膀高度）。
5. 添加子节点 **Camera2D**，
   - 设置 **Zoom** 属性：将显示器尺寸（如 1920×1080）除以目标分辨率（如 640×360），得到缩放值（约 3, 3）。你也可以直接根据游戏窗口大小调整。
   - 启用 **Smoothing** 可使摄像机移动更平滑。

## 3. 创建输入映射

为了让玩家控制角色，需要定义输入动作：

1. 打开 **项目 → 项目设置 → 输入映射**。
2. 添加以下动作，并绑定对应的键盘/手柄按键：
   - `move_left`：A / 左箭头
   - `move_right`：D / 右箭头
   - `move_up`：W / 上箭头
   - `move_down`：S / 下箭头
   - `jump`：空格
   - `shoot`：鼠标左键（或 J）

> 💡 **小提示**：可以在代码中使用 `Input.get_axis("move_left", "move_right")` 来获取水平输入值。

## 4. 创建玩家脚本

### 脚本结构（`player.gd`）

```gdscript
extends CharacterBody2D
class_name PlayerController

const SPEED = 170               # 水平移动速度
const JUMP_VELDCITY = -450      # 跳跃初速度（负值表示向上）
const GRAVITY = 1800            # 重力加速度

@onready var animated_sprite_2d: AnimatedSprite2D = $AnimatedSprite2D
@onready var shooting_point: Node2D = $Shooting_Point

var AirborneLastFrame = false   # 上一帧是否在空中（用于落地特效）
var isShooting = false          # 是否正在射击（用于动画控制）
const SHOOT_DURATION = 0.249    # 射击动作持续时间

# 玩家状态枚举
enum PlayerState { Normal, Hurt, Dead }
var currentState: PlayerState = PlayerState.Normal:
    set(new_value):
        currentState = new_value
        match currentState:
            PlayerState.Hurt:
                # 根据是否在地面播放不同受伤动画
                if is_on_floor():
                    animated_sprite_2d.play("Hit_Stand")
                else:
                    animated_sprite_2d.play("Hit_Jump")
            PlayerState.Dead:
                animated_sprite_2d.play("Die")
                set_collision_layer_value(2, false)  # 关闭玩家碰撞层，避免死后继续碰撞

var CurrentHealth
const MAX_HEALTH = 100

func _ready() -> void:
    CurrentHealth = MAX_HEALTH
    GameManeger.player = self                     # 将玩家实例存入全局管理器
    GameManeger.playerOriginalpos = position      # 记录初始位置

func _process(_delta):
    UpdateAnimation()  # 每帧更新动画（与物理处理分开）

func _physics_process(delta):
    # 应用重力
    if not is_on_floor():
        velocity.y += GRAVITY * delta
        AirborneLastFrame = true
    elif AirborneLastFrame:
        PlayLandVFX()           # 落地特效
        AirborneLastFrame = false

    # 受伤或死亡时禁止移动
    if currentState == PlayerState.Hurt or currentState == PlayerState.Dead:
        velocity.x = 0
        move_and_slide()
        return

    # 跳跃处理
    if Input.is_action_just_pressed("跳跃") and is_on_floor():
        velocity.y = JUMP_VELDCITY
        PlayJumpUpVFX()

    # 水平移动
    var direction = Input.get_axis("左", "右")
    if direction != 0:
        velocity.x = direction * SPEED
    else:
        velocity.x = 0

    # 下蹲（单向平台穿透）
    if Input.is_action_just_pressed("下") and is_on_floor():
        position.y += 3   # 轻微下移以脱离平台

    # 射击
    if Input.is_action_just_pressed("射击"):
        TryToShoot()

    move_and_slide()

func UpdateAnimation():
    if currentState == PlayerState.Dead:
        return
    elif currentState == PlayerState.Hurt:
        if animated_sprite_2d.is_playing():
            return        # 播放完受伤动画再恢复普通状态
        else:
            currentState = PlayerState.Normal

    # 根据移动方向翻转角色和射击点位置
    if velocity.x != 0:
        animated_sprite_2d.flip_h = (velocity.x < 0)
        if velocity.x < 0:
            shooting_point.position.x = -26
        else:
            shooting_point.position.x = 26

    # 地面动画
    if is_on_floor():
        if abs(velocity.x) >= 0.1:   # 奔跑
            var playingAnimationFrame = animated_sprite_2d.frame
            var playingAnimationName = animated_sprite_2d.animation

            if isShooting:
                animated_sprite_2d.play("Shoot_Run")
                # 保持与之前奔跑动画的帧同步
                if playingAnimationName == "Run":
                    animated_sprite_2d.frame = playingAnimationFrame
            else:
                # 等待射击动画播完再切换回奔跑
                if playingAnimationName == "Shoot_Run" and animated_sprite_2d.is_playing():
                    pass
                else:
                    animated_sprite_2d.play("Run")
        else:   # 站立
            if isShooting:
                animated_sprite_2d.play("Shoot_Stand")
            else:
                animated_sprite_2d.play("Idle")
    else:   # 空中
        animated_sprite_2d.play("Jump")
        if isShooting:
            animated_sprite_2d.play("Shoot_Jump")

# 特效生成封装（通过 GameManeger）
func PlayJumpUpVFX():
    var vfxToSpawn = preload("res://game/Scene/vfx_jump_up.tscn")
    GameManeger.SpawnVFX(vfxToSpawn, global_position)

func PlayLandVFX():
    var vfxToSpawn = preload("res://game/Scene/vfx_land.tscn")
    GameManeger.SpawnVFX(vfxToSpawn, global_position)

func PlayFireVFX():
    var vfxToSpawn = preload("res://game/Scene/vfx_player_fire.tscn")
    var vfxInstance = GameManeger.SpawnVFX(vfxToSpawn, shooting_point.global_position)
    if animated_sprite_2d.flip_h:
        vfxInstance.scale.x = -1   # 根据朝向翻转枪口特效

# 生成子弹
func Shoot():
    var bulletToSpawn = preload("res://game/Scene/bullet.tscn")
    var bulletInstance = GameManeger.SpawnVFX(bulletToSpawn, shooting_point.global_position)
    bulletInstance.direction = -1 if animated_sprite_2d.flip_h else 1

# 尝试射击（带冷却）
func TryToShoot():
    if isShooting:
        return
    isShooting = true
    Shoot()
    PlayFireVFX()
    await get_tree().create_timer(SHOOT_DURATION).timeout
    isShooting = false

# 受到伤害
func ApplyDamage(damage: int):
    if currentState == PlayerState.Hurt or currentState == PlayerState.Dead:
        return
    CurrentHealth -= damage
    currentState = PlayerState.Hurt
    if CurrentHealth <= 0:
        CurrentHealth = 0
        currentState = PlayerState.Dead
```

**关键点说明**：
- 状态机使用 setter 自动处理动画和碰撞层开关。
- 射击动画与移动动画通过 `isShooting` 和帧同步实现平滑过渡。
- 特效生成统一由 `GameManeger.SpawnVFX` 处理，返回实例便于进一步设置（如方向翻转）。
- 玩家受伤时会有短暂无敌（通过状态阻止再次受伤），死亡时关闭碰撞层避免死后影响。

## 5. 配置地图物理层

为了让玩家与地图碰撞，需要设置物理层：

1. **项目 → 项目设置 → 层名称 → 2D 物理**，添加自定义层名（如 `ground` 用于地面，`player` 用于玩家，`enemy` 用于敌人等）。
2. 选中 `TileMapLayer` 节点，展开 **Physics Layers**，添加一个物理层。
   - **碰撞层（Collision Layer）**：该节点属于哪一层（例如勾选 `ground` 层）。
   - **碰撞掩码（Collision Mask）**：该节点会与哪些层发生碰撞（例如只勾选 `player` 和 `enemy` 层）。
3. 在 TileSet 中为图块添加碰撞形状：选择某个图块，切换到 **物理 → 物理层 0**，添加一个 **矩形形状** 并调整大小，使其覆盖图块底部或整体。

> 🚨 **注意**：物理层数量有限（最多32层），合理规划避免浪费。

## 6. 创建玩家重置区域

### 实现逻辑
1. 在根节点下添加 **Area2D**，命名为 `RespawnArea`。
2. 添加 **CollisionShape2D** 子节点，设置形状（如矩形）覆盖需要重置的区域。
3. 设置该区域的碰撞层和掩码：
   - `RespawnArea` 的碰撞层设为 0（不需要被其他物体检测）。
   - 掩码设为 2（检测玩家所在层）。
4. 选中 `RespawnArea` 节点，在右侧 **节点** 信号面板中找到 `body_entered` 信号，点击连接，选择该节点自身并创建新方法。
5. 在信号处理函数中，调用一个全局的 **游戏管理器** 来重置玩家位置。

```gdscript
# RespawnArea 的信号处理
func _on_body_entered(body):
    if body.name == "Player":
        GameManeger.PlayerEnteredResetArea()
```

### 全局管理器（GameManeger）
在 `GameManeger.gd` 中实现重置逻辑：

```gdscript
extends Node

var player
var playerOriginalpos

func PlayerEnteredResetArea():
    player.position = playerOriginalpos

func SpawnVFX(vfxToSpawn: Resource, position: Vector2):
    var vfxInstance = vfxToSpawn.instantiate()
    vfxInstance.global_position = position
    get_tree().get_root().get_node("Root").add_child(vfxInstance)
    return vfxInstance
```


2. 在 **项目 → 项目设置 → 自动加载** 中，添加该脚本，名称设为 `GameManager`。这样游戏启动时就会自动加载，任何地方都可以使用 `GameManager.respawn_player(player)`。


> **注意**：在 `Player.gd` 的 `_ready()` 中，已将 `player` 和 `playerOriginalpos` 赋值给 `GameManeger`，因此重置时可直接使用。

## 7. 视差背景图层
为了增强视觉效果，可以添加多层视差背景（Parallax）。

### 视频做法
- 创建ParallaxBackground子节点（已改用Parallax2D）
- 静态背景
  - 创建ParallaxLayer子节点并设置属性。motion中scale属性为反向运动速度，Mirroring属性为循环
  - 创建Sprite2D节点并添加资源（ctrl拖入直接创建），Transform中设置position为(0,0)，Offset中停用中心聚焦
  - 设置ParallaxLayer节点，
- 背景动画
  - 创建ParallaxLayer子节点并设置
  - 创建AnimatedSprite2D子节点，在Sprite Frames里新建精灵帧，选择在动画表中新建精灵帧，添加帧，开启加载后自动播放
- 创建粒子效果
  - 此时，为了把粒子往画面前移动，可以新建一个ParallaxBackground节点，把层属性中CarvesLayer-Layer往大了调整。
  - 添加ParallaxLayer、AnimatedSprite2D子节点并导入粒子效果动画帧
- 添加近景画布



> **Godot 4 更新**：推荐使用 `Parallax2D` 节点替代原来的 `ParallaxBackground` 和 `ParallaxLayer`。

### 添加静态背景
1. 创建 `Parallax2D` 节点，作为背景容器。
2. 为其添加子节点 `Sprite2D`，将背景图片拖入 **Texture** 属性。
3. 调整 `Parallax2D` 的 **Scroll Scale** 属性（如 (0.5, 0.5)），使背景移动速度慢于前景，产生景深感。
4. 可勾选 **Repeat** 让背景无限循环。

### 添加动画背景（如云层）
1. 创建另一个 `Parallax2D`，添加 `AnimatedSprite2D` 子节点。
2. 为其创建 SpriteFrames，设置循环动画。
3. 调整 `Scroll Scale` 和 `Repeat`。

### 添加粒子效果（如落叶、阳光）
1. 新建 `Parallax2D` 节点，将 **Layer** 属性调大（如 5），使其显示在前景之上。
2. 添加 `AnimatedSprite2D` 或 `GPUParticles2D` 节点，导入粒子动画帧。

> 💡 **小提示**：多个 `Parallax2D` 可以共存在场景中，通过 `Layer` 属性控制前后遮挡关系。

func _physics_process(delta):

## 9. 创建敌人

### 敌人节点结构
1. 新建场景，根节点为 `CharacterBody2D`，命名为 `Enemy`，并添加 `class_name EnemyController`。
2. 添加 `CollisionShape2D` 形状（胶囊或矩形）。
3. 添加 `AnimatedSprite2D`，配置动画：`walk`、`attack`、`die`。
4. 添加两个 `RayCast2D` 节点（作为 `CollisionShape2D` 的子节点），分别用于检测前方障碍和前方悬崖,实现转向：
   - `RayCast2D_forward`：水平向前发射，检测墙壁。
   - `RayCast2D_downward`：向下前方发射，检测地面尽头。
5. 添加一个 **Node2D** 节点 `Area2D_Container`，用于统一管理攻击检测区域（可包含 `PlayerDetector` 和 `DamageCaster`）。
6. 在 `Area2D_Container` 下添加 `Area2D` 节点 `PlayerDetector`，并设置碰撞形状，掩码为 2（检测玩家层）。

### 敌人行走逻辑
- 在 `_physics_process` 中，让敌人向某个方向移动。
- 每帧检测 `RayCast2D` 是否碰撞到墙壁或悬崖边缘，若碰撞则翻转方向。
- 同时检测与玩家的碰撞：使用 `Area2D` 作为攻击检测器。

### 物理层设置
- 将敌人的碰撞层设为 3（或其他未使用层），掩码设为 1（检测地面）。
- 玩家的碰撞层设为 1，掩码设为 3（既检测地图也检测敌人）。

### 渲染顺序
- 在 `AnimatedSprite2D` 的 **Offset** 中调整 `Z Index`，使玩家穿过敌人时显示在前方（如玩家 Z=100，敌人 Z=0）。

### 实例化敌人
- 将 `Enemy` 场景保存为 `enemy.tscn`，然后在 `main.tscn` 中通过“实例化子场景”添加到关卡中。

### 敌人脚本（`Enemy.gd`）

```gdscript
extends CharacterBody2D
class_name EnemyController

const SPEED = 50
var direction = -1

@onready var animated_sprite_2d: AnimatedSprite2D = $AnimatedSprite2D
@onready var ray_cast_2d_forward: RayCast2D = $CollisionShape2D/RayCast2D_forward
@onready var ray_cast_2d_downward: RayCast2D = $CollisionShape2D/RayCast2D_downward
@onready var area_2d_container: Node2D = $Area2D_Container

var currentHealth = 100
var isDead = false
var isAttaching = false   # 是否正在攻击（控制动画）

func _process(_delta):
    UpdateAnimation()

func _physics_process(_delta):
    if isDead:
        return

    # 简单重力（避免浮空）
    if not is_on_floor():
        velocity.y = 300

    # 攻击时不能移动和转向
    if isAttaching:
        if animated_sprite_2d.is_playing() == false:
            isAttaching = false
        else:
            return

    # 转向检测：前方有墙 或 前方地面不存在
    if ray_cast_2d_forward.is_colliding() or not ray_cast_2d_downward.is_colliding():
        direction = -direction
        ray_cast_2d_forward.target_position.x = -ray_cast_2d_forward.target_position.x
        ray_cast_2d_downward.position.x = -ray_cast_2d_downward.position.x
        area_2d_container.scale.x = -direction   # 翻转攻击检测区域

    velocity.x = direction * SPEED
    move_and_slide()

func UpdateAnimation():
    if isDead:
        return
    if velocity.x != 0:
        animated_sprite_2d.flip_h = (velocity.x > 0)
    if not isAttaching:
        animated_sprite_2d.play("walk")
    elif animated_sprite_2d.animation != "attack":
        animated_sprite_2d.play("attack")

func ApplyDamage(damage: int):
    if isDead:
        return
    currentHealth -= damage
    if currentHealth <= 0:
        isDead = true
        animated_sprite_2d.play("die")
        set_collision_layer_value(3, false)   # 关闭碰撞层，避免死后继续碰撞
        await get_tree().create_timer(2).timeout
        queue_free()

func _on_area_2d_player_detector_body_entered(body: Node2D) -> void:
    isAttaching = true
```

**关键点说明**：
- 使用两个射线实现平台边缘检测，确保敌人不会掉下悬崖。
- 攻击检测区域通过 `Area2D_Container` 统一管理，翻转容器可保持攻击方向正确。
- 死亡时延迟两秒再删除，留出死亡动画播放时间。

extends Area2D

@onready var bullet_sprite_2d: Sprite2D = $Bullet_Sprite2D

const SPEED = 500
var direction = 1
const DAMAGE = 35

func _physics_process(delta: float) -> void:
    if direction < 0:
        bullet_sprite_2d.flip_h = true
    position.x += direction * SPEED * delta

func _on_body_entered(body: Node2D) -> void:
    # 生成命中特效
    var vfxToSpawn = preload("res://game/Scene/vfx_bullet_hit.tscn")

    queue_free()

## 13. 敌人攻击玩家
### 攻击检测器
1. 在敌人节点下添加 `Area2D`，命名为 `PlayerDetector`，用于检测玩家进入攻击范围。
2. 添加 `CollisionShape2D`，形状为圆形或矩形，覆盖敌人前方的攻击区域。
3. 设置该区域的碰撞层为 0（不参与碰撞），掩码为 2（检测玩家层）。
4. 连接 `body_entered` 信号，当玩家进入时执行攻击逻辑。

### 伤害传送器
- 同样创建 `DamageCaster` 的 `Area2D` 节点，用于实际造成伤害。
- 可以通过一个父节点（如 `Node2D`）统一管理这两个区域，方便跟随敌人朝向翻转。

### 实现方式
在 `Enemy` 节点下，通过 `Area2D_Container` 包含 `PlayerDetector` 区域，当玩家进入时设置 `isAttaching = true`，触发攻击动画和后续伤害判定（可进一步扩展，如使用 `DamageCaster` 区域在攻击动画帧实际造成伤害）。
### 攻击逻辑
```gdscript
func _on_player_detector_body_entered(body):
    if body.name == "Player":
        attack()

func attack():
    # 播放攻击动画
    animated_sprite.play("attack")
    # 等待攻击动画关键帧后造成伤害
    await get_tree().create_timer(0.3).timeout
    if $DamageCaster.has_overlapping_bodies():
        for body in $DamageCaster.get_overlapping_bodies():
            if body.name == "Player":
                body.take_damage(1)
```

> 💡 **提示**：若需要精确控制伤害时机，可在攻击动画关键帧处发射信号，或使用 `AnimationPlayer` 的 `call_method` 轨道调用伤害方法。
